# Code of Conduct

## Our Commitment

As a 4-person capstone team, we're committed to a respectful, collaborative, and harassment-free working environment — for each other, and for anyone else who interacts with this project (instructors, TAs, evaluators).

## Expected Behavior

- Communicate respectfully in code review, team chat, and meetings — critique code, not people.
- Give and receive feedback constructively.
- Credit each other's work accurately — Jira assignment and Smart Commits exist partly so contribution is visible and fairly attributed.
- Respect agreed-upon process (serialize points, PR review requirements, branch protection) — these protect the whole team's work, not slow anyone down arbitrarily.
- Raise disagreements directly and early, not passive-aggressively in commit messages or PR descriptions.

## Unacceptable Behavior

- Harassment, discriminatory language, or personal attacks in any project space.
- Deliberately merging over someone else's in-progress work without coordination on a known serialize point.
- Taking credit for a teammate's work.

## Scope

Applies to all project spaces: this repository (commits, PRs, issues, discussions), the team's Jira board, and team communication channels.

## Enforcement

This is a small, fixed team working under academic supervision. Concerns should be raised directly with the team first; if unresolved, escalate to the course supervisor (Syed Raza Bashir).
